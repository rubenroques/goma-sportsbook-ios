## Sync Commands

```bash
# What CI runs:

# 1. Sync catalog.json (~500 KB, instant)
rsync -avz catalog.json server:/var/www/gomaui-catalog/data/

# 2. Sync snapshots - INCREMENTAL (only changed files)
rsync -avz --checksum --delete SnapshotTests/ server:/var/www/gomaui-catalog/public/snapshots/
```

### Why --checksum Matters

Without `--checksum`, rsync uses file size + modification time. This is usually fine, but:
- CI runners may have inconsistent timestamps
- Snapshot re-generation with same content still has new timestamp

With `--checksum`, rsync computes MD5 hashes, ensuring:
- Truly identical files are never re-uploaded
- Only actual content changes trigger transfer
